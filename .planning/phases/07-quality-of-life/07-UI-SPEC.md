# Phase 7 — UI Design Contract

> Visual and interaction contract for the quality-of-life phase: home dashboard, progressive overload suggestions, CSV/PDF export, and workout history browser. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Blazor Server -- CSS custom properties) |
| Preset | not applicable |
| Component library | none (custom Razor components) |
| Charting library | BlazorApexCharts 6.1.0 (already installed, not used in this phase) |
| PDF library | QuestPDF 2026.2.4 (new) |
| CSV library | CsvHelper 33.1.0 (new) |
| Icon library | Inline SVG (project convention) |
| Font | Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif |

**Note:** This project uses a fully established dark premium design system via CSS custom properties in `wwwroot/app.css`. All new components must reference existing `--` tokens. No new design system tooling is needed.

## Interaction Contracts

### Home Dashboard

| Action | Result |
|--------|--------|
| Page load | Fetch today's scheduled workouts. If found, show first planned workout with exercise preview. If none, fetch last completed workout |
| Click "Start Session" | Navigate to `/session/{workoutId}` (same flow as WorkoutDetailDialog) |
| Click "Repeat Workout" | Create new scheduled workout for today from last completed template, then navigate to `/session/{newWorkoutId}` |
| Click "+N more exercises" | Expand to show full exercise list (client-side toggle, no DB fetch) |
| Click "Browse Templates" | Navigate to `/templates` |
| Click "Create Template" | Navigate to `/templates/new` |
| Click "Browse Exercises" | Navigate to `/exercises` |

### History Page

| Action | Result |
|--------|--------|
| Page load | Fetch 20 most recent completed sessions, fetch list of logged exercises for filter dropdown |
| Type in search | Client-side filter: match workout name or exercise name (case-insensitive contains). Debounce 300ms |
| Toggle type chip | Filter sessions to selected type(s). Multiple chips can be active. All off = show all |
| Change date range | Re-query DB with date bounds. Clear existing results and reload |
| Change exercise filter | Re-query DB for sessions containing selected exercise |
| Click a history card | Toggle expand/collapse of that card's detail panel |
| Click "Load More" | Fetch next 20 sessions, append to list. If fewer than 20 returned, hide button |

### Overload Suggestions (Session Page)

| Action | Result |
|--------|--------|
| Session starts | Query overload suggestions for all strength exercises in this session. Display suggestion card above each qualifying exercise |
| Click "Apply +{N}kg" | Update the planned weight for this exercise in the current session. Suggestion card collapses with fade-out. Toast: "Weight updated to {newWeight}kg" |
| Click "Dismiss" | Remove suggestion card with fade-out. Add exercise ID to dismissed set. No toast |
| Session ends and restarts | Dismissed suggestions reappear if criteria still met |

### Export (Analytics Page)

| Action | Result |
|--------|--------|
| Click "CSV" | Generate CSV for current time range. Download starts via JS interop. Button shows loading state during generation |
| Click "PDF" | Generate PDF for current time range. Download starts via JS interop. Button shows loading state during generation |
