```javascript
window.downloadFileFromStream = async (fileName, contentStreamReference) => {
    const arrayBuffer = await contentStreamReference.arrayBuffer();
    const blob = new Blob([arrayBuffer]);
    const url = URL.createObjectURL(blob);
    const anchorElement = document.createElement('a');
    anchorElement.href = url;
    anchorElement.download = fileName ?? '';
    anchorElement.click();
    anchorElement.remove();
    URL.revokeObjectURL(url);
}
```

**OverloadService.cs** -- Create `Services/OverloadService.cs` in namespace `BlazorApp2.Services`:

Constructor: `(IDbContextFactory<AppDbContext> contextFactory)` with `_contextFactory` field.

DTO record at top of file:
```csharp
public record OverloadSuggestion(
    int ExerciseId,
    string ExerciseName,
    double CurrentWeight,
    double SuggestedWeight,
    double Increment,
    int ConsecutiveSessions,
    int PlannedSets,
    int PlannedReps);
```
**IMPORTANT:** The record includes `PlannedSets` and `PlannedReps` fields (8 fields total). These are needed by Plan 07-05's OverloadSuggestion UI component to display "You hit {PlannedSets}x{PlannedReps} at {CurrentWeight}kg". Populate from the most recent qualifying session: `PlannedSets` = count of working sets, `PlannedReps` = PlannedReps from the first working set.

Methods:
- `public static double GetWeightIncrement(MuscleGroup muscleGroup)` -- per D-07:
  - Chest => 2.5, Back => 2.5, Shoulders => 2.5 (upper body compounds)
  - Legs => 5.0, FullBody => 5.0 (lower body compounds)
  - Arms => 1.0, Core => 1.0 (isolation)
  - _ => 2.5 (safe default)

- `public async Task<List<OverloadSuggestion>> GetSuggestionsAsync(int workoutLogId)`:
  1. Load the WorkoutLog with SetLogs and Exercises (include StrengthExercise via TPH)
  2. Get distinct strength exercise IDs from the session's SetLogs
  3. For each strength exercise:
     a. Query last 2 completed sessions for this exercise (SetLogs where IsCompleted=true, WorkoutLog.CompletedAt != null, WorkoutLogId != current, SetType == SetType.Working)
     b. Group by WorkoutLogId, order by CompletedAt descending, take 2
     c. For each session: check if ALL working sets have ActualReps >= PlannedReps AND ActualWeight >= PlannedWeight (not null)
     d. If both sessions qualify: calculate increment from exercise's MuscleGroup via GetWeightIncrement
     e. Use the PlannedWeight from the most recent session as CurrentWeight
     f. Extract PlannedSets (count of working sets) and PlannedReps (from first working set's PlannedReps) from the most recent qualifying session
     g. Create OverloadSuggestion with SuggestedWeight = CurrentWeight + Increment, PlannedSets, PlannedReps
  4. Return list of suggestions (per D-05, D-06, D-07, D-08: strength only)

**HistoryService.cs** -- Create `Services/HistoryService.cs` in namespace `BlazorApp2.Services`:

Constructor: `(IDbContextFactory<AppDbContext> contextFactory)` with `_contextFactory` field.

DTO records at top:
```csharp
public record HistorySession(
    int WorkoutLogId,
    DateTime CompletedAt,
    string WorkoutName,
    string WorkoutType,
    int ExerciseCount,
    int DurationMinutes,
    double TotalVolume,
    double? TotalDistance,
    int? Rpe,
    string? Notes,
    int? TemplateId,
    List<HistoryExerciseDetail> Exercises);

public record HistoryExerciseDetail(
    string ExerciseName,
    bool IsStrength,
    List<HistorySetDetail> Sets,
    HistoryEnduranceDetail? Endurance);

public record HistorySetDetail(int SetNumber, double? Weight, int? Reps, string SetType);
public record HistoryEnduranceDetail(double? Distance, int? DurationSeconds, double? Pace);
```
**IMPORTANT:** The `HistorySession` record includes a `TemplateId` (int?) field (12 fields total). This is needed by Plan 07-03's Repeat Workout flow (D-03) to schedule a new workout from the same template. Populate from `WorkoutLog.ScheduledWorkout.WorkoutTemplateId`. If the original workout was ad-hoc (no template), this will be null -- the Repeat flow should then schedule an ad-hoc workout with the same name.

Methods:
- `public async Task<List<HistorySession>> GetCompletedSessionsAsync(DateTime? dateStart, DateTime? dateEnd, int? exerciseId, int skip, int take)`:
  Query WorkoutLogs where CompletedAt != null, apply date range filter, apply exerciseId filter (sessions containing that exercise in SetLogs or EnduranceLogs). Include ScheduledWorkout with WorkoutTemplate for workout name, include SetLogs with Exercise, include EnduranceLogs with Exercise. Order by CompletedAt descending. Skip/Take for pagination. Map to HistorySession DTOs. WorkoutName = template name ?? ad-hoc name ?? "Untitled". WorkoutType = determine from exercise types (has strength + endurance = "Mixed", only strength = "Strength", only endurance = "Endurance"). DurationMinutes = (CompletedAt - StartedAt).TotalMinutes rounded. TotalVolume = sum of (ActualWeight * ActualReps) for completed working sets. TotalDistance = sum of ActualDistance for completed endurance logs. TemplateId = WorkoutLog.ScheduledWorkout.WorkoutTemplateId. Exercises list populated with details for the expand view.

- `public async Task<int> GetTotalCountAsync(DateTime? dateStart, DateTime? dateEnd, int? exerciseId)`:
  Same filters as above but returns count only (for "Load More (N remaining)" display).

- `public async Task<List<(int Id, string Name)>> GetLoggedExercisesAsync()`:
  Query distinct exercises that appear in completed SetLogs or EnduranceLogs. Return (Id, Name) tuples for filter dropdown.

- `public async Task<HistorySession?> GetLastCompletedWorkoutAsync()`:
  Query most recent WorkoutLog where CompletedAt != null, with full includes. Map to HistorySession (including TemplateId). Used for home screen "last completed" (per D-03). Returns actual logged weights (not template targets).

- `public async Task<ScheduledWorkout?> GetTodaysScheduledWorkoutAsync()`:
  Query ScheduledWorkouts for today (DateTime.UtcNow.Date), Status == Planned, include WorkoutTemplate with Items and Exercises, ordered by Id, take first. Per QOL-01.

- `public async Task<ScheduledWorkout?> GetTomorrowsScheduledWorkoutAsync()`:
  Same pattern but for tomorrow's date (DateTime.UtcNow.Date.AddDays(1)). Per D-02.
