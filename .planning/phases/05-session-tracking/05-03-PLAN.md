Build the core session tracking UI: the Session page (landing state with today's workouts and active state with session header, progress bar, and exercise list) and the SessionExerciseItem component (collapsible exercise with strength set rows, endurance inputs, previous performance drawer, and exercise status buttons). This is where users spend their entire workout logging time.

Purpose: This is the heart of the session tracking feature -- the interface users interact with during every workout. Handles both strength and endurance exercises with type-appropriate inputs, inline previous performance, and exercise-level completion status.
Output: Session.razor (page + code-behind + CSS), SessionExerciseItem.razor (component + CSS)

**Session.razor** -- Create the page with two routes and two states:

1. Routes: `@page "/session"` and `@page "/session/{ScheduledWorkoutId:int}"`
2. Add `@rendermode InteractiveServer`
3. Add `@inject SessionService SessionService`, `@inject NavigationManager NavigationManager`

**Landing state** (when no ScheduledWorkoutId parameter or no active session):
- On page init, call `SessionService.GetIncompleteSessionIdAsync()`. If returns a value, call `NavigationManager.NavigateTo($"/session/{incompleteId}", replace: true)` per D-15
- Otherwise, call `SessionService.GetTodaysWorkoutsAsync()` to show today's scheduled workouts
- Render a list of today's workouts as cards showing: workout name, template exercise count, "Start Session" button
- If no workouts: empty state with heading "No workouts today", body "Schedule a workout from the calendar, or start an ad-hoc session.", "Go to Calendar" link button
- "Start Session" button calls `SessionService.StartSessionAsync(workout.Id)` then navigates to `/session/{workout.Id}`

**Active session state** (when ScheduledWorkoutId is set):
- On init, call `SessionService.StartSessionAsync(ScheduledWorkoutId)` which handles both new and resume cases
- If resumed (WorkoutLog had sets with IsCompleted=true already), show toast "Session resumed -- picking up where you left off"
- Store WorkoutLog in component state

**Session header** (sticky, 56px height):
- Left: Back arrow SVG (16x16, `<path d="M19 12H5M5 12l7 7M5 12l7-7"/>` in 24x24 viewBox) wrapped in 36x36 button -- clicking opens abandon confirmation (wired in Plan 05, for now navigate to /calendar)
- Center: Workout name in heading font, truncated with ellipsis
- Right: Elapsed time display (mm:ss or h:mm:ss), font-variant-numeric: tabular-nums
- Far right: "Abandon Session" ghost text button in error color (wired in Plan 05 for confirm dialog, for now calls AbandonSessionAsync and navigates)

**Progress bar** (below header, also sticky):
- Segmented bar with one segment per exercise (strength exercises counted once per exercise, endurance counted as 1)
- Track exercise IDs from WorkoutLog: group SetLogs by ExerciseId + EnduranceLogs by ExerciseId = ordered list of unique exercise IDs
- Each segment: flex: 1, 8px height, border-radius: var(--radius-full)
- Segment colors based on exerciseStatuses dictionary:
  - Pending: `var(--color-progress-empty)`
  - Complete: `var(--color-accent)` (filled)
  - Partial: `var(--color-exercise-partial-text)` (amber)
  - Skipped: `var(--color-text-tertiary)` at opacity 0.4
- Label: "{N}/{M} exercises" right-aligned below bar

**Exercise list:**
- Render `<SessionExerciseItem>` for each unique exercise in the workout (ordered by first SetLog/EnduranceLog position)
- Pass parameters: exercise entity, set logs for this exercise, endurance log for this exercise, completion status, isExpanded, position number
- Track `expandedExerciseId` (only one expanded at a time per D-08). First pending exercise expanded by default.
- EventCallbacks from SessionExerciseItem: OnSetCompleted, OnSetUncompleted, OnEnduranceSaved, OnSetAdded, OnStatusChanged, OnSetTypeChanged

**"Finish Session" button** at bottom:
- Visible only when all exercises have a status (Complete, Partial, or Skipped)
- Uses pulseGlow animation to draw attention
- For now, clicking shows the SessionSummary (wired in Plan 05; for now just call FinishSessionAsync and navigate)
