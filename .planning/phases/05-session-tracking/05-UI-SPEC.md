# Phase 5 -- UI Design Contract

> Visual and interaction contract for real-time session tracking. Start a workout from a scheduled entry, log sets with type-appropriate inputs, view previous performance, mark exercise completion status, and finish with RPE + notes. Incremental persistence ensures circuit-death resilience. Continues the premium dark-mode design system established in Phase 2 and carried through Phase 3/4. Generated by gsd-ui-researcher.

## Visual Hierarchy

**Session page focal points:**
- **Primary focal point:** The currently expanded exercise panel. Elevated background, accent left border, set rows with large tap targets. This is where the user spends 90% of their time during a session.
- **Secondary focal point:** The progress bar at the top of the page. Segmented fill showing exercise completion status provides constant context of session progress.
- **Tertiary focal point:** The session header showing the workout name and elapsed time. Sticky at top for always-visible context.
- **Summary focal point:** When the session ends, the summary screen takes full focus with volume stats, RPE slider, and notes -- the final action point before closing out.

## Interaction Contracts

### Session Page (Landing -- `/session`)

| Interaction | Behavior |
|-------------|----------|
| Page load | Check for incomplete session (StartedAt set, CompletedAt null). If found, auto-redirect to `/session/{id}` per D-15. Otherwise, show today's scheduled workouts. |
| Click "Start Session" on a workout | Create WorkoutLog with StartedAt=now, snapshot planned targets into SetLog/EnduranceLog rows. Navigate to `/session/{scheduledWorkoutId}`. |
| No workouts today | Show empty state with link to calendar. |

### Session Page (Active -- `/session/{id}`)

| Interaction | Behavior |
|-------------|----------|
| Page load | Load WorkoutLog with all SetLogs/EnduranceLogs. Expand first exercise with status "Pending". Show progress bar reflecting current completion state. Start elapsed time counter. |
| Page load (resumed session) | Same as above, but pre-fill actual values from previously saved data. Show "Session resumed" toast. |
| Click collapsed exercise | Collapse currently expanded exercise, expand clicked one. Smooth transition via exerciseExpand/exerciseCollapse. |
| Click "Previous" drawer | Toggle previous performance drawer open/closed. Load last 3 session records for this exercise on first open (on-demand, not eager). |
| Edit weight/reps in set row | Update value in memory. Value is persisted when checkmark is tapped. |
| Tap checkmark on set row | Mark set as completed (D-13). Immediately persist SetLog to database (IsCompleted=true, ActualWeight, ActualReps). Checkmark animates with `setCheckIn`. Row background transitions to `--color-set-complete-bg`. |
| Tap completed checkmark | Un-complete the set. IsCompleted=false persisted. Row returns to default state. Inputs become editable again. |
| Tap set type label | Cycle through: Working -> Warm-up -> Failure -> Drop -> Working (D-02). Persist type change immediately. |
| Tap "+ Add Set" | Append new SetLog row below last set. Pre-fill weight from previous set's actual weight (D-03). Set number increments. Animate in with `rowSlideIn`. |
| Fill endurance fields | Update values in memory. Pace auto-calculates from distance/duration (D-06). |
| Tap "Complete" on exercise | Mark as Complete. Persist endurance data if endurance exercise. Collapse exercise (200ms delay). Auto-expand next pending exercise. Update progress bar segment. |
| Tap "Partial" on exercise | Mark as Partial. Same collapse/expand behavior. Progress bar segment shows amber. |
| Tap "Skip Exercise" on exercise | Mark as Skipped. Collapse, expand next. Progress bar segment shows gray at 0.4 opacity. |
| Tap "Abandon Session" in header | Open AbandonSessionDialog. |
| Confirm abandon | Set CompletedAt=now on WorkoutLog. Mark ScheduledWorkout as Completed (with partial data, D-16). Navigate to `/calendar`. Toast "Session complete". |
| All exercises have status | Show "Finish Session" button at bottom of exercise list with `pulseGlow` animation to draw attention. |
| Tap "Finish Session" | Navigate to SessionSummary overlay. |
| Back arrow in header | Same as "Abandon Session" flow -- open confirmation dialog since session is in progress. |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | All interactive elements reachable via Tab. Enter/Space to activate buttons, toggle set type, complete sets. Arrow Up/Down to move between set rows within an expanded exercise. Escape to close summary overlay or abandon dialog. |
| Focus indicators | 2px solid `--color-accent`, offset 2px on all focusable elements (consistent with Phase 2/3/4) |
| ARIA roles | `role="progressbar"` + `aria-valuenow` + `aria-valuemin="0"` + `aria-valuemax="{total}"` on progress bar. `aria-expanded` on exercise collapse/expand. `role="dialog"` + `aria-modal="true"` on abandon dialog and summary overlay. `aria-label` on icon-only buttons (checkmark, set type cycle, back arrow). `role="slider"` + `aria-valuenow` + `aria-valuemin="1"` + `aria-valuemax="10"` on RPE slider. `aria-pressed` on exercise status buttons. `role="group"` + `aria-label="Set {N}"` on each set row. |
| Screen reader | Exercise items announce: name + type + status. Set rows announce: "Set {N}, {type}, {weight}kg x {reps}". Checkmark announces: "Complete set" / "Undo complete". Status buttons announce: "Mark exercise as complete/partial/skipped". Progress bar announces: "{N} of {M} exercises complete". RPE slider announces: "Rate of perceived exertion, {value}, {descriptor}". |
| Reduced motion | All animations gated behind `@media (prefers-reduced-motion: no-preference)` |
| Touch targets | Minimum 44px for all touch-interactive elements. Checkmark button 36px diameter + 4px padding = 44px hit area. Set type label minimum 32px height with padding to 44px. Number inputs 32px height on desktop, 44px on mobile. |
| Live regions | `aria-live="polite"` on progress bar label for exercise completion updates. Toast uses existing live region pattern. |
