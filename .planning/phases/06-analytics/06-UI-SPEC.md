# Phase 6 — UI Design Contract

> Visual and interaction contract for the analytics dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Blazor Server -- CSS custom properties) |
| Preset | not applicable |
| Component library | none (custom Razor components) |
| Charting library | BlazorApexCharts 6.1.0 |
| Icon library | Inline SVG (project convention) |
| Font | Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif |

**Note:** This project uses a fully established dark premium design system via CSS custom properties in `wwwroot/app.css`. All new components must reference existing `--` tokens. No new design system tooling is needed.

### ApexCharts Color Palette

Configure in `AddApexCharts()` global options:

```
Colors: ["#60A5FA", "#34D399", "#7C5CFC", "#FBBF24", "#EF4444"]
```

Order: strength blue, endurance green, accent purple, warning amber, error red. This ensures the first two series in any chart map to the established strength/endurance convention.

Accent reserved for: active tab indicator border-bottom, active time range selector background, PR badge glow, "New PR" toast accent, page header underline accent

## Layout Specification

### Page Structure

```
+--------------------------------------------------+
|  Analytics                    [ 4W  8W  12W  All ]|
+--------------------------------------------------+
| [Overview] [Strength] [Endurance] [PRs]           |
+--------------------------------------------------+
| TAB CONTENT                                       |
|                                                   |
+--------------------------------------------------+
```

### Page Header

- Page title: "Analytics" at `--font-size-heading`, weight 600
- Time range selector: aligned right, pill-shaped segmented control (same pattern as Calendar view-toggle)
- Default selection: 4W (per D-14)
- Options: 4W / 8W / 12W / All (per D-15)

### Tab Bar

- Four tabs: **Overview**, **Strength**, **Endurance**, **PRs** (per D-05)
- Tab style: pill-shaped segmented control matching Calendar view-toggle pattern
- Active tab: `--color-accent-muted` background, `--color-accent` text, 1px accent-tinted border
- Inactive tab: transparent background, `--color-text-secondary` text
- Tab bar sits below page header with `--space-lg` gap

### Tab Content Area

Top padding `--space-lg` from tab bar. Max-width 960px, centered.

## Tab Content Contracts

### Overview Tab

**KPI Cards Row (per D-06, D-07):**

4-column grid (2-column on mobile):

| Card | Label | Value Example | Subtitle Example |
|------|-------|---------------|------------------|
| Sessions This Week | SESSIONS THIS WEEK | 4 | "of 5 planned" |
| Current Streak | CURRENT STREAK | 12 days | "longest: 18 days" |
| Total Volume | TOTAL VOLUME | 8,240 kg | "this period" |
| Latest PR | LATEST PR | Bench Press | "72.5 kg -- 2 days ago" |

**KPI Card Visual:**
- Background: `--color-bg-secondary`
- Border: 1px solid `--color-border-subtle`
- Border-radius: `--radius-md` (12px)
- Padding: `--space-md` (16px)
- Value: `--font-size-display` (28px), weight 600, `--color-text-primary`
- Label: `--font-size-label` (12px), weight 400, uppercase, `--color-text-secondary`, letter-spacing 0.04em
- Subtitle: `--font-size-label` (12px), weight 400, `--color-text-tertiary`
- Layout: centered text, label above value, subtitle below value

**Charts below KPI cards:**
1. **Weekly Volume** (bar chart) -- total weight lifted per week in blue (#60A5FA)
2. **Adherence** (grouped bar chart) -- planned (outline/ghost) vs completed (filled green) per week

Chart container: `--color-bg-secondary` background, 1px `--color-border-subtle` border, `--radius-md` border-radius, `--space-md` padding.

### Strength Tab

**Exercise Selector (per D-08):**
- Dropdown at top: "All Exercises" default, lists strength exercises the user has logged
- Style: same as existing `.filter-select` pattern from Exercises page
- Selecting an exercise filters all charts to that single exercise

**Charts:**
1. **Volume per Week** (bar chart) -- total sets x weight in blue (#60A5FA)
2. **Max Weight Trend** (line chart) -- heaviest weight per week, blue line with dot markers
3. **Estimated 1RM Trend** (line chart) -- best weekly e1RM via Epley, purple (#7C5CFC) line

### Endurance Tab

**Exercise Selector (per D-08):**
- Dropdown at top: "All Activities" default, lists endurance exercises the user has logged
- Same `.filter-select` style

**Charts:**
1. **Distance per Week** (bar chart) -- total km per week in green (#34D399)
2. **Pace Trend** (line chart) -- average pace (min/km) per week, green line with dot markers
   - Y-axis inverted: lower pace = faster = better (lower is up)

### PRs Tab

**PR Record Book (per D-12, D-13):**

Layout: list of PR entries grouped by exercise name.

Each exercise section:
- Exercise name as section header at `--font-size-body` (15px), weight 600
- PR entries as rows beneath:

| PR Type | Value | Date |
|---------|-------|------|
| Weight PR | 80.0 kg | Mar 15, 2026 |
| Rep PR | 12 reps @ 60 kg | Mar 10, 2026 |
| Est. 1RM | 92.0 kg | Mar 15, 2026 |

For endurance exercises:

| PR Type | Value | Date |
|---------|-------|------|
| Pace PR | 4:32 min/km | Mar 12, 2026 |
| Distance PR | 15.2 km | Mar 8, 2026 |

**PR Entry Visual:**
- Row background: `--color-bg-secondary`
- Border: 1px solid `--color-border-subtle`
- Border-radius: `--radius-sm` (8px)
- Padding: `--space-sm` (8px) vertical, `--space-md` (16px) horizontal
- PR type label: `--font-size-label`, `--color-text-secondary`, uppercase
- PR value: `--font-size-body`, weight 600, `--color-text-primary`
- PR date: `--font-size-label`, `--color-text-tertiary`
- PR type badge: small pill with type-appropriate color (strength blue, endurance green)

**PR Timeline (per D-13):**
- Below the record book, a line chart showing PR achievements over time
- X-axis: dates, Y-axis: PR values (normalized or per-exercise)
- Scatter plot markers at each PR date with tooltip showing exercise name + value

**Filtering:** Only exercises the user has actually logged appear (per D-12). If the user only runs, they see only running PRs.

## Interaction Contracts

### Time Range Selector

| Action | Result |
|--------|--------|
| Click 4W / 8W / 12W / All | All charts in the current tab reload with data from the selected range. Active button gets accent styling. |
| Default state on page load | 4W is selected (per D-14) |

### Tab Navigation

| Action | Result |
|--------|--------|
| Click a tab | Tab content switches immediately (no page navigation). Active tab gets accent styling. Data loads for the selected tab. |
| Default state on page load | Overview tab is active |

### Chart Interactions (per D-02)

| Action | Result |
|--------|--------|
| Hover on bar/point | Tooltip appears showing exact value (e.g., "Week of Mar 9: 2,450 kg") |
| Empty/skipped week | Bar shows as zero-height with week label still present on X-axis (per D-17) |

### Exercise Drill-Down (per D-08)

| Action | Result |
|--------|--------|
| Select exercise from dropdown | Charts filter to show only that exercise's data. Dropdown shows exercise name. |
| Select "All Exercises" / "All Activities" | Charts show aggregated data across all exercises of that type |

### PR Detection Toast (per D-09)

| Action | Result |
|--------|--------|
| Session finishes with new PR | Toast notification: "New PR! [Exercise Name] -- [Value]" with accent glow |
| Multiple new PRs in one session | One toast per PR, stacked or sequential with 3-second intervals |

## Empty States

### No Sessions Logged (new user)

**Applies to:** All tabs when no WorkoutLog records exist.

```
[Chart icon placeholder -- muted SVG]

No workout data yet

Complete your first session to see analytics here.
Start by scheduling a workout on the Calendar.
```

- Icon: 48x48 inline SVG chart icon in `--color-text-tertiary`
- Heading: `--font-size-body` (15px), weight 600, `--color-text-secondary`
- Body: `--font-size-body` (15px), weight 400, `--color-text-tertiary`
- "Calendar" is a text link in `--color-accent` linking to `/calendar`
- Centered in tab content area with `--space-2xl` top padding

### No Data in Selected Range

**Applies to:** When selected time range (e.g., 4W) has no sessions but other ranges do.

```
No data in the last 4 weeks

Try expanding the time range or check back after your next session.
```

- Same visual treatment as above but without the chart icon
- Time range label dynamically matches selection ("4 weeks", "8 weeks", "12 weeks")

### No PRs Yet

**Applies to:** PRs tab when no PersonalRecord entries exist.

```
No personal records yet

PRs are automatically detected when you finish a session.
Keep training and your records will appear here.
```

### No Exercises of Type

**Applies to:** Strength tab when user has only logged endurance sessions (and vice versa).

```
No strength sessions logged yet

Complete a strength workout to see your lifting analytics.
```

## Responsive Behavior

### Desktop (>=768px)

- KPI cards: 4-column grid
- Charts: full width within max-width 960px container
- Tab bar and time range selector: single row, space-between
- PR entries: table-style rows

### Mobile (<768px)

- KPI cards: 2-column grid
- Charts: full width with horizontal scroll if needed (ApexCharts handles responsive)
- Tab bar: horizontally scrollable if tabs overflow
- Time range selector: moves below page title (stacked layout)
- PR entries: stacked card layout

### Breakpoint

Single breakpoint at 768px matching the Calendar page's responsive approach (desktop 7-col grid vs mobile single-column).

All animations respect `prefers-reduced-motion: no-preference` (project convention).

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Tab navigation | `role="tablist"` on tab container, `role="tab"` on each tab button, `aria-selected="true/false"`, `role="tabpanel"` on content |
| Chart alt text | Each `<ApexChart>` container gets `aria-label` describing the chart (e.g., "Weekly volume bar chart showing total weight lifted per week") |
| Time range buttons | `aria-pressed="true/false"` on active/inactive buttons |
| KPI cards | Semantic HTML with label/value structure, no additional ARIA needed |
| Color contrast | All text meets WCAG AA: primary text #F0F0F5 on #0A0A0F = 18.3:1, secondary text rgba(240,240,245,0.60) on #0A0A0F = 9.4:1 |
| Keyboard navigation | Tab through tabs with arrow keys, Enter/Space to activate. Tab to time range buttons. |

## ApexCharts Theme Configuration

Global configuration in `Program.cs` via `AddApexCharts()`:

```
Theme.Mode = "dark"
Chart.Background = "transparent"
Colors = ["#60A5FA", "#34D399", "#7C5CFC", "#FBBF24", "#EF4444"]
Tooltip.Enabled = true
```

Per-chart overrides via individual `ApexChartOptions<T>` instances (never shared between charts).

Chart containers use `--color-bg-secondary` background with `--radius-md` border-radius so the transparent chart floats on the card surface.
