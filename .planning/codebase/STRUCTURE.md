# Codebase Structure

## Directory Purposes

**Components/:**
- Purpose: Central location for all Razor component definitions
- Contains: Root component, router, layout components, page components, shared components
- Key files: `App.razor`, `Routes.razor`, `_Imports.razor`
- Structure: Organized by function (Layout for shared UI, Pages for routable pages)

**Components/Layout/:**
- Purpose: Shared layout and structural components used across pages
- Contains: MainLayout (primary page wrapper), ReconnectModal (connection recovery UI)
- Key files: `MainLayout.razor`, `ReconnectModal.razor`, `ReconnectModal.razor.js`
- Inheritance: MainLayout extends LayoutComponentBase for use as page layout

**Components/Pages/:**
- Purpose: Routable pages that appear at specific URLs
- Contains: Page components with @page directives defining routes
- Key files: `Home.razor` (homepage), `Error.razor` (error page), `NotFound.razor` (404 page)
- Pattern: Each file represents one route; Router scans these components for route definitions

**Properties/:**
- Purpose: Project configuration and runtime metadata
- Contains: Launch settings, project properties, assembly information
- Key files: `launchSettings.json` (defines http and https debug profiles)

**wwwroot/:**
- Purpose: Static web assets served directly to browser without processing
- Contains: CSS files, JavaScript, images, fonts
- Key files: `app.css` (global styles referenced in App.razor)
- Framework: Blazor framework generates additional CSS file (`BlazorApp2.styles.css`) at build time

## Key File Locations

**Entry Points:**

- `Program.cs`: Application startup - configures services, middleware, and starts HTTP server. Runs on server before any client requests.
- `Components/App.razor`: Root component - defines HTML document structure, loads stylesheets and framework. First component rendered in browser.
- `Components/Routes.razor`: Router configuration - determines which page component to render based on URL.

**Configuration:**

- `appsettings.json`: Shared configuration across all environments (logging levels, allowed hosts)
- `appsettings.Development.json`: Development-only overrides (currently empty)
- `Properties/launchSettings.json`: Debug profile configuration (HTTP port 5235, HTTPS port 7189)
- `BlazorApp2.csproj`: Project metadata (target framework net10.0, nullable reference types enabled)

**Core Logic:**

- `Components/Layout/MainLayout.razor`: Shared page wrapper and error boundary UI
- `Components/Pages/Error.razor`: Global error page handler with request ID display and C# code-behind
- `Components/Pages/Home.razor`: Welcome page at route `/`
- `Components/Pages/NotFound.razor`: 404 page for unmatched routes

**Styling:**

- `wwwroot/app.css`: Global stylesheet for application
- Error UI styles in MainLayout.razor (inline CSS for error boundary)
- ReconnectModal styles in ReconnectModal.razor (inline CSS for dialog)

## Naming Conventions

**Files:**

- PascalCase with .razor extension: `App.razor`, `MainLayout.razor`, `Home.razor`
- Code-behind files: `ComponentName.razor.cs` (not currently present) or inline `@code` blocks
- Supporting JavaScript: `ComponentName.razor.js` (used for ReconnectModal)
- Stylesheet: `ComponentName.razor.css` (generated by build system for scoped styles)

**Directories:**

- PascalCase for feature grouping: `Components`, `Layout`, `Pages`, `Properties`
- Organize by function/feature, not by type

**C# Classes and Namespaces:**

- Namespace: `BlazorApp2` (root), `BlazorApp2.Components` (for component code-behind)
- Namespaces auto-imported via `_Imports.razor` in Components directory

**Routes:**

- Lowercase with forward slash: `@page "/"`, `@page "/not-found"`, `@page "/Error"`
- Error page uses `/Error` (uppercase) by convention

## Where to Add New Code

**New Feature (with UI):**
- Primary component: `Components/Pages/FeatureName.razor` (if routable) or `Components/FeatureName.razor` (if shared)
- Business logic: `Services/FeatureService.cs` (create if needed; register in Program.cs)
- Styling: `wwwroot/feature-name.css` or scoped in component with `ComponentName.razor.css`

**New Shared Component/Module:**
- Implementation: `Components/SomeComponent.razor` (general-purpose shared components) or `Components/Layout/SomeComponent.razor` (if used in layout)
- Code-behind: `Components/SomeComponent.razor.cs` (if complex logic needed)

**New Page:**
- Implementation: `Components/Pages/PageName.razor` with `@page "/pagename"` directive
- Layout: Inherits from MainLayout by default via Router configuration

**Utilities/Services:**
- Shared helpers: Create `Services/` directory if not present; register in Program.cs as service dependency
- Extension methods: Create `Extensions/` directory
- Constants: Create `Constants.cs` or feature-specific files

**Styling/Static Assets:**
- Global styles: `wwwroot/app.css`
- Component-specific styles: `wwwroot/components/componentname.css` or `Components/ComponentName.razor.css` (scoped)
- Layout styles: `wwwroot/layout.css`

## Special Directories

**bin/ and obj/:**
- Purpose: Build output and compiler cache
- Generated: Yes
- Committed: No (in .gitignore)

**.vs/:**
- Purpose: Visual Studio project-specific settings and cache
- Generated: Yes
- Committed: No

**wwwroot/:**
- Purpose: Static web assets served directly by HTTP middleware
- Generated: Partially (Blazor generates scoped style files during build)
- Committed: Yes (original files), No (generated files)

**Properties/:**
- Purpose: Project metadata and runtime configuration
- Generated: No (created at project initialization)
- Committed: Yes
